The ASU CampusLabs Engage API
=========================
The Engage API enables developers to programmatically obtain and modify information about the organization. Available 
endpoints include:
* Attendees
* Curriculums
* Events
* Experiences
* Finance Accounts/Request/Transactions
* Memberships
* Organizations
* Positions
* Test (does not contain live info)
* Users

Currently, the API is not available to CodeDevils (or any FSO) due to potential issues with [FERPA Compliance](https://www2.ed.gov/policy/gen/guid/fpco/ferpa/index.html),
but this knowledge article will serve as a baseline for when CodeDevils obtains access to the API.

## Documentation

Documentation for the Engage API can be found [here](https://asu.campuslabs.com/engage/api/docs). You can also find out
more about the Engage API on the [CampusLabs help center](https://engagesupport.campuslabs.com/hc/en-us/articles/360027628671-Using-the-Engage-API).

## The Swagger UI

The [Swagger UI](https://swagger.io/) is an interface used to document and test API endpoints. CampusLabs uses Swagger
to document its endpoints and enables testing with a valid JWT token. The [Swagger interface for ASU](https://asu.campuslabs.com/engage/api/swagger/ui/index)
shows all the endpoints available for use through ASU.
